    sol-dnn-v1 (Speech Overlap Detection)

    Description

    Speech OverLap Detection (SOL) is a technique used to pinpoint instances of concurrent speech, enabling the identification of situations where a live speaker is talking simultaneously with external speech sources, like a TV, radio, or another speaker.

    Domains

    • multi-v1
      • Multi-condition domain trained on speech overlapped push-to-talk (PTT), telephony, and distant-microphone speech.

    Inputs

    An audio file or buffer and (optional) identifier and (optional) regions to be scored. If no regions are specific the entire audio file or buffer will be scored.

    Outputs

    The SOL plugin generates region scores that offer insights into speech overlap occurrences. In its initial step, this plugin computes a log-likelihood ratio (LLR) score for distinguishing single speaker speech from overlapping speech. This score is computed for every 10ms frame within the input audio segment. These frame scores are then further processed internally to determine speech overlap regions, with a degree of padding and interpolation applied.

    The outcome of the plugin's operation is a set of region scores that indicate the time intervals, expressed in seconds, where instances of speech overlap have been identified in the audio recording.

    When the plugin is called upon to provide region scores, the conversion from frame scores to region scores is managed internally. This conversion involves applying a specific threshold to the scores. Frames that form contiguous sequences and surpass the threshold are transformed into timestamp pairs, signifying the initiation and conclusion of each identified region. Subsequently, these regions are expanded in duration by subtracting a specified padding value (commonly around 0.5 seconds or lower) from the region's start time and adding the same padding value to the region's end time. To illustrate, if an initial region spans from 10 seconds to 20 seconds and a 1-second padding is applied, the resultant 'padded' region will span from 9 to 21 seconds. In cases where padded regions overlap, they are merged into an extended single region.

    An example of SOL region scores:

        test_1.wav 0.301 16.759 overlap 0.00000000
    test_1.wav 30.034 47.210 overlap 0.00000000

    The final number in this format is the threshold employed to achieve those regions (0.0 by default).

    Functionality (Traits)

    The functions of this plugin are defined by its Traits and implemented API messages. A list of these Traits is below, along with the corresponding API messages for each. Click the message name below to be brought to additional implementation details below.

    • REGION_SCORER – Score all submitted audio, returning labeled regions within the submitted audio, where each region represents timestamp boundaries where speech was detected.
      • RegionScorerRequest

    Compatibility

    OLIVE 5.4+

    Limitations

    Any known or potential limitations with this plugin are listed below.

    False Alarms in Broadcast recordings

    The SOL plugin has the potential to false alarms, particularly when applied to broadcast recordings. This is attributed to the extensive training of the model using audio broadcast recordings as background data.

    Minimum Audio Length

    A minimum waveform duration of 0.31 seconds is required to produce a detection. However, for meaningful detections, it is advisable to utilize extended audio recordings exceeding a duration of 5 minutes.

    Comments

    Speech Region Padding

    When performing region scoring with this plugin, a 0.3 second padding is added to the start and end of each detected speech region, in order to reduce edge-effects for human listening, and to otherwise smooth the output regions of SAD.

    Global Options

    The following options are available to this plugin, adjustable in the plugin's configuration file; plugin_sad_config.py.

    Option Name Description Default Expected Range
    sol_threshold Detection threshold: Higher value results in less detections being output. Increase the threshold value to reduce the number and duration of false detections. Reduce the threshold value if there are too many missed segments. 0.0 -5.0 to 5.0
    score_offset Score offset value: Used to this value from the frame scores. Higher value results in less detections being output. Increase the score offset value to reduce the number and duration of false detections. Reduce the threshold value if there are too many missed segments. 0.0 -5.0 to 5.0
    sol_filter Smoothing filter: Used to smooth the raw frame scores. Increase the value if there is too many sparse detections and to achieve longer continuous island of segments (detections). For example: smoothing filter value 101 indicates 1 second of window filter is being used to smooth out raw frame scores. 3001 101 to 5001
